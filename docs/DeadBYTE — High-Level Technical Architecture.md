  4.2 Interface Contracts

  ┌─────────────────────────────────────────────────────────────────────────────────┐
  │                          INTERFACE CONTRACTS                                    │
  ├─────────────────────────────────────────────────────────────────────────────────┤
  │                                                                                 │
  │  COMMAND CONTRACT (UI → Backend)                                               │
  │  ───────────────────────────────                                               │
  │                                                                                 │
  │   Every command MUST:                                                          │
  │   ✓ Accept strongly-typed input parameters                                     │
  │   ✓ Validate all inputs before processing                                      │
  │   ✓ Return a Result<T, Error> type                                             │
  │   ✓ Never panic (all errors handled)                                           │
  │   ✓ Be async (non-blocking)                                                    │
  │   ✓ Emit progress events for long operations                                   │
  │                                                                                 │
  │   Example:                                                                     │
  │   ┌─────────────────────────────────────────────────────────────────────────┐  │
  │   │                                                                         │  │
  │   │  #[tauri::command]                                                      │  │
  │   │  async fn scan_directory(                                               │  │
  │   │      window: tauri::Window,                                             │  │
  │   │      path: String,                                                      │  │
  │   │      options: ScanOptions                                               │  │
  │   │  ) -> Result<ScanResult, CommandError> {                                │  │
  │   │      // Validate                                                        │  │
  │   │      let path = validate_path(&path)?;                                  │  │
  │   │                                                                         │  │
  │   │      // Execute with progress                                           │  │
  │   │      let result = analyzer_engine::scan(path, options, |progress| {    │  │
  │   │          window.emit("scan-progress", progress).ok();                   │  │
  │   │      }).await?;                                                         │  │
  │   │                                                                         │  │
  │   │      Ok(result)                                                         │  │
  │   │  }                                                                      │  │
  │   │                                                                         │  │
  │   └─────────────────────────────────────────────────────────────────────────┘  │
  │                                                                                 │
  │  SERVICE CONTRACT (Engine → Service)                                           │
  │  ────────────────────────────────────                                          │
  │                                                                                 │
  │   Every service MUST:                                                          │
  │   ✓ Be stateless (no internal mutable state)                                   │
  │   ✓ Handle its own errors (never propagate raw system errors)                  │
  │   ✓ Log significant operations                                                 │
  │   ✓ Be thread-safe (Send + Sync)                                               │
  │   ✓ Provide cancellation support where applicable                              │
  │                                                                                 │
  │   Services MUST NOT:                                                           │
  │   ✗ Call other services directly (go through engine)                           │
  │   ✗ Make business decisions                                                    │
  │   ✗ Access configuration directly (receive as parameter)                       │
  │                                                                                 │
  │                                                                                 │
  │  EVENT CONTRACT (Backend → UI)                                                 │
  │  ─────────────────────────────                                                 │
  │                                                                                 │
  │   All events MUST:                                                             │
  │   ✓ Have a unique event name                                                   │
  │   ✓ Carry a typed payload                                                      │
  │   ✓ Be documented                                                              │
  │                                                                                 │
  │   Event categories:                                                            │
  │   • progress-*    : Operation progress updates                                 │
  │   • status-*      : Status changes                                             │
  │   • result-*      : Operation results                                          │
